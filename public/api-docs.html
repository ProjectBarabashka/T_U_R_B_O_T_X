Quick Start #quickstart

Get your first API response in under 2 minutes.

1
Get an API Key
Contact us at @turbotx_support on Telegram or email to receive your API key. Keys look like: ttx_live_...
2
Verify your key
Make a ping request to confirm authentication works.
cURL 
curl "https://acelerat.vercel.app/api/v1?method=ping" \
  -H "Authorization: Bearer ttx_live_YOUR_KEY"
3
Check a transaction
Get full status, fee analysis, and acceleration advice for any unconfirmed TX.
JavaScript 
const res = await fetch(
  'https://acelerat.vercel.app/api/v1?method=status&txid=YOUR_TXID',
  { headers: { 'Authorization': 'Bearer ttx_live_YOUR_KEY' } }
);
const data = await res.json();
console.log(data.status);          // 'mempool' | 'confirmed'
console.log(data.feeRate);         // e.g. 12 sat/vB
console.log(data.accelerationAdvice); // { action: 'boost', urgency: 'medium' }
4
Accelerate a stuck TX
Submit the transaction to 30 broadcast channels covering ~88% of Bitcoin hashrate.
JavaScript 
const res = await fetch('https://acelerat.vercel.app/api/v1?method=accelerate', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer ttx_live_YOUR_KEY',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    txid: 'YOUR_64_CHAR_TXID',
    plan: 'premium',           // 'free' | 'premium'
    webhookUrl: 'https://yourapp.com/webhook', // optional
  })
});
const data = await res.json();
console.log(data.ok);            // true
console.log(data.hashrateReach); // "~88%"
Authentication #auth

All API requests require an API key. Three methods are supported — Bearer header is recommended.

✦ Recommended — Authorization Header
Authorization: Bearer ttx_live_...
X-API-Key Header
X-API-Key: ttx_live_...
Query Parameter (not for production)
?apikey=ttx_live_...
⚠ Never expose keys client-side
Keep keys in env vars or a backend proxy

POST ?method=accelerate Submit TX for acceleration

Broadcast a transaction to all 30 channels — 8 hex nodes and 22 mining pool accelerators covering ~88% of Bitcoin hashrate. Pro/Partner keys automatically use Premium plan.

ParameterTypeRequiredDescription
txid string required 64-character transaction ID
plan string optional "free" or "premium". Default: free. Pro/Partner keys always get premium.
webhookUrl string optional URL to receive a POST callback when acceleration starts
⚡
Premium plan broadcasts to MARA Slipstream (private mempool bypass) and all 22 mining pool accelerators in 8 waves over up to 4 hours.

GET ?method=cpfp&txid=<64hex> CPFP fee calculator

Calculate exact fee needed for a Child-Pays-For-Parent transaction. Returns sat amounts, USD cost, and wallet-specific instructions for Electrum, Sparrow, and BlueWallet.

ParameterTypeRequiredDescription
txid string required Parent transaction ID
outputIndex integer optional Which output to use for CPFP (default: 0, auto-selects largest)
target string optional eco | std | fast (default: fast)
GET ?method=rbf&txid=<64hex> RBF fee bump calculator

Check if RBF is enabled and calculate the fee needed to replace the transaction. Compares RBF vs CPFP cost.

GET ?method=health[&verbose=1] Service & channel health

Live health check of all 30 broadcast channels. Add ?verbose=1 to get per-channel ping times.

Error Codes #errors

All errors return a JSON body with ok: false and an error string.

HTTPCode / ReasonResolution
200 ok: false inside body TX not found, method returned empty result
400 Invalid txid / method Check txid is 64 hex chars, check method name
401 Unauthorized API key missing or invalid — check Authorization header
429 Rate limit exceeded Slow down or upgrade tier. See resetAt in response.
500 Internal server error Upstream Bitcoin node error. Retry with exponential backoff.
Error Response Body
{
  "ok": false,
  "error": "Rate limit exceeded (per_minute)",
  "limit": 30,
  "resetAt": 1710000000000,
  "upgradeUrl": "https://acelerat.vercel.app/api-docs#pricing"
}
